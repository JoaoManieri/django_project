Criando uma ambiente virtual

O módulo venv fornece suporte para a criação de “ambientes virtuais” leves com seus próprios diretórios de sites, opcionalmente isolados dos diretórios de sites do sistema. Cada ambiente virtual tem seu próprio binário Python (que corresponde à versão do binário que foi usado para criar esse ambiente) e pode ter seu próprio conjunto independente de pacotes Python instalados em seus diretórios de site.


comando para criar ambiente virtual. 

 $ python3 -m venv ./venv

Para ativar o ambiente virtual basta dar esse comando no console. 

 $ venv\Scripts\activate

Instalando o Django em seu ambiente virtual

O framework django é uma ferramenta que possibilita desenvolver apps web como apis e paginas inteiras. Neste caso, como pretendemos fazer o deply no heroku, temos que, até a data deste de hoje (18/06/2022) temos que utilizar a versao 3.2.13, porém o django se encontra na versao 4.x.x.


comando para baixar o pacote django. 

 $ pip install django==3.2.13

comando para baixar o pacote django rest. 

 $ pip install djangorestframework

Criando arquivos do Django framework.

Para criar o projeto Django é necessario um pacote de arquivo já prontos, para isso você precisa escolher o nome desta pasta, no meu caso escolhi o nome setup. Usamos o "." ao fim para indicar ao console que queremos criar a os arquivos na pasta anterior.


comando para criar arquivos. 

 $ django-admin startproject setup .

Pronto, agora é só testar localmente seu primeiro site.

Para testar basta rodar o seguinte comando


comando para rodar o site localmente. 

 $ python manage.py runserver

Apos rodar o comando deve aparecer um link, e se vc clicar neste link sera redirecionado ao seu site.
